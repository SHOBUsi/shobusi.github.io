Two-Tier PKI Design & Implementation

AWS • Windows Server 2019 • Root CA + Issuing CA • TLS • Wireshark

Objective

Build a secure two-tier Public Key Infrastructure (PKI) to enable certificate issuance and enforce TLS encryption, demonstrating enterprise trust architecture and cloud security fundamentals.

Architecture

  • Root CA (offline / highly protected concept)
  • Issuing CA for day-to-day certificate issuance
  • TLS-enabled web server using issued certificates
  • Cloud deployment using AWS compute instances

Implementation (high level)

  • Configured CA roles and certificate templates
  • Issued and managed certificates
  • Enabled HTTPS/TLS on the web service
  • Validated encrypted traffic via packet capture

Security Outcome

  • Reduced MITM risk with enforced TLS
  • Better trust control using tiered CA model
  • Demonstrated verification with evidence (pcap)
  • Clear documentation for repeatable setup

UK Job Relevance (keywords)

Cloud Security (AWS) PKI / Certificates TLS / HTTPS Network Traffic Analysis Documentation

This case study demonstrates secure design thinking (trust boundaries, certificate lifecycle) and validation skills (Wireshark evidence) that are common in UK security engineering and analyst roles.
